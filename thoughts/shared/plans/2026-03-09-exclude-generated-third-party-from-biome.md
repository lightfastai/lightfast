# Exclude Generated/Third-Party Code from Biome Linting

## Overview

Follow the next-forge pattern of excluding generated and third-party code from Biome linting via `!` negation patterns in `files.includes`. This prevents unnecessary formatting churn on code we don't own (shadcn/ui components, generated types) and removes biome-ignore suppressions added in the Group 1 rule enablement PR.

## Current State Analysis

- Single root `biome.jsonc` lints the entire monorepo
- The Group 1 rule enablement PR (`899a1c9eb`) reformatted all 50 shadcn/ui components (semicolons, import order, Tailwind class sorting, attribute sorting) and added 7 `biome-ignore` suppressions — all unnecessary churn on third-party generated code
- `apps/www/src/lib/generated/` contains only `lissajous-paths.ts` (auto-generated SVG path data) — this file should live in `packages/ui/src/lib/` instead
- `vendor/cms/basehub-types.d.ts` is auto-generated by BaseHub SDK

### Key Discoveries:
- next-forge excludes: `packages/design-system/components/ui`, `packages/design-system/lib`, `packages/design-system/hooks`, `packages/cms/basehub-types.d.ts` — exact same pattern we need
- 58 files were changed in directories that should be excluded (50 shadcn/ui components, 3 hooks, 4 lib files, 1 generated type file)
- `packages/ui` exports map uses `"./lib/*": "./src/lib/*.ts"` — so moving lissajous there works with existing exports

## Desired End State

- `biome.jsonc` excludes generated/third-party directories
- `lissajous-paths.ts` lives in `packages/ui/src/lib/` (excluded from biome)
- All formatting churn and biome-ignore suppressions in excluded directories are reverted
- `pnpm check` and `pnpm typecheck` pass cleanly

## What We're NOT Doing

- Excluding vendor packages (they're first-party wrappers, should be linted)
- Excluding `packages/ui/src/components/ai-elements/` (custom components we maintain)
- Adding per-package biome configs (staying with single root config pattern)
- Changing any biome rules

## Implementation Approach

Single phase — all changes are local, low-risk, and interdependent.

## Phase 1: Exclude Generated/Third-Party Code and Move Lissajous

### Overview
Update biome.jsonc exclusions, move lissajous-paths.ts to the UI package, revert all formatting churn in excluded directories, and update imports.

### Changes Required:

#### 1. Move lissajous-paths.ts to UI package
**From**: `apps/www/src/lib/generated/lissajous-paths.ts`
**To**: `packages/ui/src/lib/lissajous-paths.ts`

```bash
git mv apps/www/src/lib/generated/lissajous-paths.ts packages/ui/src/lib/lissajous-paths.ts
```

#### 2. Update biome.jsonc exclusions
**File**: `biome.jsonc`
**Changes**: Replace current includes with next-forge-style exclusions

```jsonc
"files": {
  "includes": [
    "**/*",
    "!.claude",
    "!packages/ui/src/components/ui",
    "!packages/ui/src/hooks",
    "!packages/ui/src/lib",
    "!vendor/cms/basehub-types.d.ts"
  ]
}
```

Note: `!apps/www/src/lib/generated` removed since the directory will be empty after moving lissajous-paths.ts.

#### 3. Update import in app-footer.tsx
**File**: `apps/www/src/components/app-footer.tsx`
**Changes**: Update import path

```typescript
// Before
import { LISSAJOUS_PATHS } from "~/lib/generated/lissajous-paths";

// After
import { LISSAJOUS_PATHS } from "@repo/ui/lib/lissajous-paths";
```

#### 4. Update output path in render-lissajous.ts
**File**: `packages/console-remotion/src/render-lissajous.ts`
**Changes**: Update generated file output path

```typescript
// Before
"../../../apps/www/src/lib/generated/lissajous-paths.ts"

// After
"../../../packages/ui/src/lib/lissajous-paths.ts"
```

#### 5. Revert formatting churn in excluded directories
**Directories**: `packages/ui/src/components/ui/`, `packages/ui/src/hooks/`, `packages/ui/src/lib/`, `vendor/cms/basehub-types.d.ts`
**Changes**: Restore main branch versions of all files in excluded directories

```bash
git checkout main -- packages/ui/src/components/ui/ packages/ui/src/hooks/ packages/ui/src/lib/ vendor/cms/basehub-types.d.ts
```

Note: `packages/ui/src/lib/` revert is safe because the lissajous file was moved there via `git mv` (from a different location), not modified in-place.

#### 6. Clean up empty generated directory
**Directory**: `apps/www/src/lib/generated/`
**Changes**: Remove empty directory if no other files remain

```bash
rmdir apps/www/src/lib/generated/
```
