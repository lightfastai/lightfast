# Blog Drafts

Blog post drafts generated by `/create_blog` command.

## Workflow

1. **Create**: `/create_blog` generates AEO-optimized drafts here
2. **Validate**: `/validate_blog` checks drafts against style guides
3. **Publish**: `/publish_blog` pushes to BaseHub CMS

## File Format

Drafts use YAML frontmatter with markdown body:

```yaml
---
title: "Post Title"
slug: "post-slug"
publishedAt: "YYYY-MM-DD"
category: "technology" | "company" | "product"
excerpt: "Short summary for listings"
tldr: "80-100 word summary for AI citation"
seo:
  metaDescription: "150-160 char meta description"
  focusKeyword: "primary keyword"
author: "jeevanpillay"
_internal:
  status: draft | published
  generated: "ISO timestamp"
---

## TL;DR

[Content...]
```

## Categories

- **technology**: Technical deep-dives, architecture, research (800-1,500 words)
- **company**: Funding, partnerships, events, hiring (300-800 words)
- **product**: Feature launches, updates, tutorials (500-1,000 words)
