⏺ Three realistic options:

  1. Unsigned closed beta — forge.config.ts:14-41 already gates
  osxSign/osxNotarize on env presence, so an unsigned build will pack cleanly;
  users get a Gatekeeper "unidentified developer" warning that they bypass with
  right-click → Open. The catch: Squirrel.Mac auto-update requires a signed app,
   so users would be on manual .dmg reinstalls until Apple is sorted. Cheapest
  path to "people are running it today" — fine for Discord/early-adopter beta,
  painful for a public launch.
  2. Start Apple enrollment in parallel — $99/yr; individual Apple Dev
  enrollment is usually same-day to ~24h, but a D-U-N-S-verified org account
  ("Lightfast AI Inc") takes 1-2 weeks. If the legal entity isn't load-bearing
  yet, enroll personally now and migrate the cert later. This is the only path
  to a real auto-updating macOS release.
  3. Ship Windows first / stay web-only — MakerSquirrel is configured but also
  unsigned (no Windows EV cert wired up); Windows SmartScreen is more forgiving
  than Gatekeeper but still warns. Probably not worth the detour given
  lightfast.ai already runs.

  My take: option 1 + 2 in parallel — cut an unsigned
  @lightfast/desktop@0.1.0-beta.1 for a tight closed beta this week (requires
  either a desktop-release-unsigned.yml variant or making the Apple steps if:

  so you can drop the unsigned hack within ~2 weeks. The main risk is people on
  the unsigned build won't auto-receive the signed v0.1.0 — they'd need a
  one-time manual reinstall.
