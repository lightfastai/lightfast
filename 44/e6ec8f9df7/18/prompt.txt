## Analysis: GitHub Binding Flow

### Overview

A Lightfast organization has a permanent 1:1 binding to a GitHub organization, stored in `lightfast_github_org_bindings`. When an org is created, the user is immediately redirected to connect GitHub. The OAuth popup flow runs through the platform service, which resolves the GitHub account, enforces binding constraints, and writes the binding record. Every access to the org workspace checks binding status — no active binding means a redirect back to the connect screen.

**Table: `lightfast_github_org_bindings`**
Defined at `/Users/jeevanpillay/Code/@lightfastai/lightfast/db/app/src/schema/tables/github-org-bindings.ts`

Key columns:
- `clerk_org_id` — unique index, maps to Clerk org (one row per Lightfast org)
- `github_org_id` — unique index, maps to GitHub account ID as string (one row per GitHub org)
