## Key Architectural Changes with Neural Memory

### 1. Dual Processing Pipelines

The architecture now features two parallel processing pipelines:

**Knowledge Layer Pipeline (Existing)**
- Processes documents, files, and structured content
- Creates chunks with embeddings
- Focuses on static, durable information

**Neural Memory Pipeline (New)**
- Extracts observations from events
- Generates multi-view embeddings
- Creates summaries and updates profiles
- Tracks temporal states

### 2. Enhanced Event System

New event types for neural memory:
- `observation.capture` - Atomic moments detected
- `summary.generate` - Clustering and rollup triggers
- `profile.update` - Actor model updates
- `state.transition` - Temporal state changes
- `memory.consolidate` - Memory optimization

### 3. Unified Storage with Segregated Namespaces

**PostgreSQL**: Separate table sets for knowledge and neural layers
- Knowledge: `workspace_documents`, `workspace_chunks`
- Neural: `workspace_neural_observations`, `workspace_neural_summaries`, `workspace_actor_profiles`, `workspace_temporal_states`

**Pinecone**: Namespaced vector stores
- `workspace_{id}_docs` - Document and chunk embeddings
- `workspace_{id}_neural` - Observation, summary, and profile embeddings

### 4. Cross-Enrichment Layer

New processing stage that connects both layers:
- Links observations to source documents
- Enriches documents with temporal context
- Adds actor metadata to content
- Connects decisions to implementations

### 5. Intelligent Retrieval Router

Enhanced router with multiple modes:
- **Knowledge Mode**: Traditional document/chunk search
- **Neural Mode**: Observation/summary/profile search
- **Hybrid Mode**: Fusion of both layers
- **Temporal Mode**: Time-aware with state tracking

### 6. Fusion Scoring Engine

Sophisticated scoring that combines:
- Knowledge relevance (0.30)
- Neural match (0.25)
- Temporal relevance (0.15)
- Actor similarity (0.15)
- Recency decay (0.10)
- Source importance (0.05)

## Performance Implications

### Benefits
- **Richer Context**: Captures "who", "when", "why" beyond just "what"
- **Better Personalization**: Actor profiles enable user-specific ranking
- **Temporal Intelligence**: Understands evolution and current state
- **Improved Recall**: Multi-view embeddings catch different query intents

### Trade-offs
- **Storage**: ~2x storage for dual layers (mitigated by compression)
- **Processing**: Additional pipeline (parallelized for minimal latency)
- **Complexity**: More components to monitor (offset by better insights)

### Optimization Strategies
1. **Selective Observation**: Only capture significant moments
2. **Batch Processing**: Summaries and profiles update periodically
3. **Smart Caching**: Frequently accessed profiles in Redis
4. **Progressive Enhancement**: Neural features optional per query

## Summary

The Neural Memory integration transforms the architecture from a document-centric system to a comprehensive memory system that understands:

1. **What happened** (observations)
2. **Who was involved** (profiles)
3. **When it occurred** (temporal states)
4. **Why it matters** (summaries)
5. **How it connects** (cross-enrichment)

This creates a living, evolving knowledge base that improves with use and provides increasingly intelligent retrieval over time.
