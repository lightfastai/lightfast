### Navigate + extract

Use `--auto-connect` on every call; it discovers port 9222 automatically.

```bash
# 1. Go to a timeline URL
agent-browser --auto-connect open https://x.com/home
agent-browser --auto-connect wait --load networkidle

# 2. Extract the first N posts as structured JSON (always wrap JS in an IIFE —
#    eval context persists across calls, so bare `const` collides on re-runs).
agent-browser --auto-connect eval --stdin <<'JS'
(() => {
  const posts = Array.from(document.querySelectorAll('article[data-testid="tweet"]')).slice(0, 15);
  return JSON.stringify(posts.map(a => ({
    author_handle: a.querySelector('[data-testid="User-Name"] a[href^="/"]')?.getAttribute('href') || null,
    post_url: a.querySelector('time')?.closest('a')?.getAttribute('href') || null,
    posted: a.querySelector('time')?.getAttribute('datetime') || null,
    text: (a.querySelector('[data-testid="tweetText"]')?.innerText || '').slice(0, 300),
    likes: a.querySelector('[data-testid="like"]')?.innerText || '',
    replies: a.querySelector('[data-testid="reply"]')?.innerText || '',
    reposts: a.querySelector('[data-testid="retweet"]')?.innerText || '',
  })), null, 2);
})();
JS

# 3. For a promising author, visit their profile and extract bio + follower count
agent-browser --auto-connect open https://x.com/<handle>
agent-browser --auto-connect wait --load networkidle
agent-browser --auto-connect eval --stdin <<'JS'
(() => {
  const bio = document.querySelector('[data-testid="UserDescription"]')?.innerText || '';
  const name = document.querySelector('[data-testid="UserName"]')?.innerText || '';
  const stats = {};
  document.querySelectorAll('a').forEach(a => {
    const href = a.getAttribute('href') || '';
    if (href.endsWith('/verified_followers') || href.endsWith('/followers')) stats.followers = a.innerText.trim();
    if (href.endsWith('/following')) stats.following = a.innerText.trim();
  });
  return JSON.stringify({ name, bio: bio.slice(0, 400), stats });
})();
JS
```

### Rules

- Always `--auto-connect`. Never pass `--session-name` (bundled-Chromium construct; ignored on CDP and gets blocked by x.com).
- Always `wait --load networkidle` after an `open` before extracting — x.com loads content async.
- Always wrap eval JS in an IIFE `(() => { ... })()` and return a string (JSON.stringify).
- Read-only. No `click`, `fill`, `type`, `press` on anything that could post/reply/like/follow/DM.
- If the first extract returns empty arrays, the cookies may have expired silently. Stop and report `auth: stale`.

## Author quality bar

**Auto-include:**
- 500+ followers AND bio mentions: founder, CTO, eng, builds, ships, AI, agents, dev tools, infra, researcher
- Verified blue + technical content (skip influencer / general-business blue checks)
- Anyone @lightfastai already follows or who follows back (`"Follows you"` badge visible on profile)

**Auto-skip:**
- <500 followers (unless bio is bullseye ICP — *founder of <agent infra startup>*, *eng at <known AI lab>*)
- Bio is generic ("entrepreneur", "investor", "AI enthusiast" with no specifics)
- Last 5 posts are all hashtag-stuffed promo
- Account created in last 90 days with thread-bot posting cadence
- Engagement looks bot-amplified: 100+ likes / 0 replies, identical reply patterns

## Post quality bar

**Engage:**
- Genuine question or working-out-loud thinking about agents/memory/infra/MCP
- Strong opinion grounded in shipped work (not airport-takeoff threads)
- Bug report, frustration, or unmet need in the dev-tool stack — Lightfast-shaped problem
- Replies on launch / changelog threads from Anthropic, Cursor, Vercel, Linear, Sentry

**Skip:**
- Pure promotion / launch announcement of an MCP-aggregator / vibes product
- Politics, drama, hot-takes detached from work
- Posts older than 6 hours unless mega-thread
- Anything with 50+ replies — reply gets buried

## Reply rubric — Lightfast voice

**Voice:** technical, curious, lowercase ok, dry. Never starts with *"great point"* / *"this"* / *"so true"*. Never pitches Lightfast unprompted. Never bait questions. 1–10 words, hard cap.

**Good replies:**
- *"persistent state across runs is the unlock"* (peer founder shipped a primitive)
- *"goal-fit signals beat recency every time"* (someone working out memory ranking)
- *"citations on every memory writeback helps"* (someone raising memory-poisoning)
- *"same — cold tool calls especially"* (someone complaining about Claude Code latency)

**Bad replies:**
- *"great point! we're building exactly this"* (sycophantic pitch)
- *"have you tried Lightfast?"* (pitch, ignores the post)
- *"agents need memory to scale, totally agree"* (generic)
- *"interesting take — what do you think about XYZ?"* (bait question)

## Scoring + dedupe

Per candidate:
- `icp_fit` (1–5): author matches Lightfast's ICP
- `resonance` (1–5): post hits Lightfast's value prop
- `engagement_signal` (1–5): fresh + getting traction (not buried, not bot-amplified)
- `priority_score` = `icp_fit + resonance + engagement_signal` (max 15)

Dedupe by `author_handle` — one post per author per run.

Cutoff: only return candidates with `priority_score ≥ 10`. Don't pad with sub-10s; surface fewer, higher-quality candidates.
