Lightfast is the memory layer for software teams and AI agents. This guide walks through the current setup path: email-based account access, organization creation, and native app sign-in.

## Step 1 — Sign in with email

Go to <a href="https://lightfast.ai/sign-in">lightfast.ai</a> and enter your email address. Lightfast sends a one-time verification code to that address. Enter the code to complete sign-in.

New users can create an account from the sign-up page. Invited users should use the invitation link from email, which accepts the team invitation directly after verification.

## Step 2 — Create an organization

Create an organization for your team. An organization is the shared workspace where team membership and product access are managed.

Choose a short, stable slug for the organization, such as `acme-platform` or `frontend`. You can rename the organization later from team settings.

## Step 3 — Invite teammates

Add the people who should have access to the organization. Invitations are accepted through email, so teammates do not need a social account or an external identity provider to join.

## Step 4 — Sign in from native tools

For local developer workflows, use `lightfast login` or the desktop sign-in button. Both use Clerk OAuth Authorization Code + PKCE, open the Lightfast web app for organization selection, and store an org-bound native session locally.

Native login does not create Lightfast API keys.
